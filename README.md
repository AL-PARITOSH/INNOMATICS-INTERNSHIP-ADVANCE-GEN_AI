# Logic Building Task – 1

## Overview

This assignment demonstrates the implementation of fundamental Python programming concepts through practical scenarios such as authentication systems, academic result analysis, message validation, and system log monitoring.

### Key Focus Areas

## Problem Statements and Implementation

### 1. Login Validation

**Objective:** Validate user login credentials.

**Requirements:**
- Accept username and password using `input()`
- Compare with predefined credentials
- Display:
  - "Login Successful" if both match
  - "Invalid Credentials" otherwise

**Concepts Used:**
- Conditional statements  
- Logical operators  
- User input handling  

---

### 2. Student Result Analysis

**Objective:** Analyze student marks and determine pass/fail statistics.

**Requirements:**
- Accept marks using `map()`
- A student passes if marks ≥ 50
- Count:
  - Total passed students
  - Total failed students
- Display passed and failed students with their respective marks

**Concepts Used:**
- Lists  
- Loops  
- Indexing  
- Conditional logic  
- `map()` function  

---

### 3. Username Cleaning and Standardization

**Objective:** Standardize user-provided names.

**Requirements:**
- Accept multiple names as input
- Remove extra spaces
- Convert names to lowercase
- Display cleaned names separated by commas

**Concepts Used:**
- `strip()`  
- `lower()`  
- `split()`  
- `join()`  
- `map()`  

---

### 4. Message Size Analyzer

**Objective:** Analyze message length and detect long messages.

**Requirements:**
- Accept multiple messages
- Print the length of each message
- Flag messages longer than 10 characters

**Concepts Used:**
- `len()`  
- Loops  
- Conditional statements  

---

### 5. System Log Error Detection

**Objective:** Detect and count "ERROR" messages from system logs.

**Requirements:**
- Accept log entries as input
- Count occurrences of "ERROR"
- Display total error count

**Concepts Used:**
- String comparison  
- Iteration  
- Counting logic
